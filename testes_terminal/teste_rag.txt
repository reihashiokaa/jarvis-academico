============================================================
OBSERVAÇÕES IMPORTANTES SOBRE OS TESTES DO RAG
============================================================

O RAG depende dos arquivos que estiverem dentro da pasta:

data/materiais

Antes de testar, confira se essa pasta existe e se tem pelo menos alguns arquivos .txt ou .pdf.

Exemplos de arquivos aceitos:

- .txt
- .pdf

A primeira vez que as funções de embeddings forem executadas, pode demorar bastante, porque o modelo de embeddings precisa ser carregado.

O modelo usado no projeto é:

sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2

Se o modelo ainda não estiver baixado localmente, a primeira execução pode tentar baixar o modelo da internet.

Se a API do Gemma estiver lenta, os testes finais do RAG também podem demorar ou retornar mensagem de timeout, porque a função final envia o contexto para a LLM responder.

17. TESTAR A FUNÇÃO consultar_material_rag()
============================================================

Essa função executa o fluxo completo do RAG.

Ela é uma função FINAL.

Isso significa que ela será cadastrada depois no tools.py como ferramenta disponível para a LLM.

Ela faz:

1. recebe a pergunta do usuário;
2. recupera os chunks relevantes;
3. formata os chunks como contexto;
4. envia pergunta + contexto para o Gemma;
5. recebe a resposta gerada;
6. devolve a resposta final.

Comando:

python -c "from src.rag import consultar_material_rag; print(consultar_material_rag('Explique o padrão Decorator com base nos materiais'))"

Resultado esperado:

Uma resposta em linguagem natural, baseada nos materiais.

Resultado aceitável se a API estiver instável:

O modelo demorou demais para responder e a chamada atingiu o tempo limite. Tente novamente em alguns instantes.

O que esse teste verifica:

- se o RAG completo funciona isoladamente;
- se a função final retorna uma resposta pronta;
- se a função pode ser usada depois pelo tools.py.

============================================================
PROBLEMAS COMUNS
============================================================

1. Erro dizendo que pypdf não existe:

Instale as dependências:

pip install pypdf

ou:

pip install -r requirements.txt


2. Erro dizendo que sentence_transformers não existe:

Instale:

pip install sentence-transformers

ou:

pip install -r requirements.txt


3. Erro dizendo que sklearn não existe:

Instale:

pip install scikit-learn

ou:

pip install -r requirements.txt


4. O primeiro teste de embeddings demorou muito:

Isso é normal na primeira execução, porque o modelo pode estar sendo carregado ou baixado.


5. O RAG retornou mensagem de timeout:

Isso significa que a chamada ao Gemma demorou demais.
Não necessariamente é erro do RAG.
A função chamar_llm() já trata esse caso e devolve uma mensagem amigável.


6. O RAG retornou resposta ruim:

Verifique primeiro os chunks recuperados:

python -c "from src.rag import recuperar_chunks_relevantes; chunks = recuperar_chunks_relevantes('sua pergunta aqui', quantidade=3); [print(f\"Fonte: {c['nome_arquivo']} | Similaridade: {c['similaridade']}\\n{c['texto'][:500]}\\n\") for c in chunks]"

Se os chunks recuperados forem ruins, o problema está na recuperação.

Se os chunks recuperados forem bons, mas a resposta for ruim, o problema provavelmente está no prompt enviado ao Gemma.


7. O RAG não encontrou documentos:

Confira se a pasta existe:

data/materiais

E confira se há arquivos .txt ou .pdf dentro dela.
