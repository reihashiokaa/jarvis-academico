============================================================
01. TESTAR A FUNÇÃO registrar_chamada_ferramenta() do arquivo logger.py
============================================================

Essa função registra uma chamada de ferramenta no arquivo logs/tool_calls.jsonl.

Ela é uma função intermediária.

Isso significa que ela ajuda o sistema por dentro, mas não será chamada diretamente pelo usuário final.

Ela grava no log:

- data e hora da chamada;
- nome da ferramenta chamada;
- entrada recebida pela ferramenta;
- saída devolvida pela ferramenta.

Comando para registrar um log de teste:

python -c "from src.logger import registrar_chamada_ferramenta; registrar_chamada_ferramenta('teste_ferramenta', {'mensagem': 'entrada de teste'}, 'saída de teste'); print('Log registrado com sucesso.')"

Comando para visualizar o arquivo de logs no CMD:

type logs\tool_calls.jsonl

Caso o terminal esteja mostrando acentos de forma errada:

chcp 65001, para fazê-lo usar UTF-8, que é a codificação correta para acentos.

02. TESTAR A FUNÇÃO carregar_ferramentas_disponiveis() do arquivlo tools.py
============================================================

Essa função monta um dicionário com todas as ferramentas disponíveis no sistema.

Ela é uma função intermediária.

Isso significa que ela ajuda o sistema por dentro, mas não será chamada diretamente pelo usuário final.

Ela liga o nome textual da ferramenta à função Python real.

Exemplo:

"consultar_tarefas" → função consultar_tarefas()

Esse teste mostra apenas os nomes das ferramentas cadastradas.

Comando:

python -c "from src.tools import carregar_ferramentas_disponiveis; ferramentas = carregar_ferramentas_disponiveis(); print(list(ferramentas.keys()))"

============================================================
03. TESTAR A FUNÇÃO recuperar_ferramenta_por_nome() do arquivo tools.py
============================================================

Essa função recebe o nome textual de uma ferramenta e tenta recuperar a função Python correspondente.

Ela é uma função intermediária.

Isso significa que ela ajuda o sistema por dentro, mas não será chamada diretamente pelo usuário final.

Ela será importante no fluxo de tool calling, porque a LLM escolherá ferramentas pelo nome.

Exemplo:

"consultar_tarefas"

A função deve encontrar a função Python real:

consultar_tarefas()

Comando para testar com uma ferramenta existente:

python -c "from src.tools import recuperar_ferramenta_por_nome; ferramenta = recuperar_ferramenta_por_nome('consultar_tarefas'); print(ferramenta)"

Comando para testar com uma ferramenta inexistente:

python -c "from src.tools import recuperar_ferramenta_por_nome; ferramenta = recuperar_ferramenta_por_nome('ferramenta_inexistente'); print(ferramenta)"

============================================================
04. TESTAR A FUNÇÃO executar_ferramenta()
============================================================

Essa função recebe o nome de uma ferramenta e uma entrada em formato de dicionário.

Ela recupera a função Python correspondente, executa essa função, registra a chamada no arquivo logs/tool_calls.jsonl e devolve a saída.

Ela é uma função intermediária.

Isso significa que ela ajuda o sistema por dentro, mas não será chamada diretamente pelo usuário final.

Ela será uma peça central no futuro fluxo de tool calling.

Comando para testar uma ferramenta sem entrada:

python -c "from src.tools import executar_ferramenta; print(executar_ferramenta('consultar_tarefas', {}))"

Comando para testar uma ferramenta com entrada:

python -c "from src.tools import executar_ferramenta; print(executar_ferramenta('adicionar_tarefa', {'titulo': 'Testar tools.py', 'descricao': 'Verificar se executar_ferramenta chama adicionar_tarefa corretamente'}))"

Comando para testar uma ferramenta inexistente:

python -c "from src.tools import executar_ferramenta; print(executar_ferramenta('ferramenta_inexistente', {}))"

Comando para visualizar o log no CMD:

type logs\tool_calls.jsonl
