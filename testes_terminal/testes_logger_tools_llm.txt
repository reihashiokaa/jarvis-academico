01. TESTAR A FUNÇÃO registrar_chamada_ferramenta() do arquivo logger.py
============================================================

Essa função registra uma chamada de ferramenta no arquivo logs/tool_calls.jsonl.

Ela é uma função intermediária.

Isso significa que ela ajuda o sistema por dentro, mas não será chamada diretamente pelo usuário final.

Ela grava no log:

- data e hora da chamada;
- nome da ferramenta chamada;
- entrada recebida pela ferramenta;
- saída devolvida pela ferramenta.

Comando para registrar um log de teste:

python -c "from src.logger import registrar_chamada_ferramenta; registrar_chamada_ferramenta('teste_ferramenta', {'mensagem': 'entrada de teste'}, 'saída de teste'); print('Log registrado com sucesso.')"

Comando para visualizar o arquivo de logs no CMD:

type logs\tool_calls.jsonl

Caso o terminal esteja mostrando acentos de forma errada:

chcp 65001, para fazê-lo usar UTF-8, que é a codificação correta para acentos.

06. TESTAR A FUNÇÃO interpretar_decisao_ferramenta() do arquivo llm_client.py
============================================================

Essa função recebe um texto que deveria estar em formato JSON e tenta transformar esse texto em um dicionário Python.

Ela é uma função intermediária.

Isso significa que ela ajuda o sistema por dentro, mas não será chamada diretamente pelo usuário final.

Ela será usada depois da função decidir_chamada_ferramenta(), porque o Gemma devolve a decisão como texto, e o Python precisa interpretar esse texto como dados.

O formato esperado da decisão é:

{
  "usar_ferramenta": true,
  "nome_ferramenta": "consultar_tarefas",
  "entrada": {}
}

Comando para testar com JSON válido:

python -c "from src.llm_client import interpretar_decisao_ferramenta; texto = '{\"usar_ferramenta\": true, \"nome_ferramenta\": \"consultar_tarefas\", \"entrada\": {}}'; print(interpretar_decisao_ferramenta(texto))"

Comando para testar com JSON inválido:

python -c "from src.llm_client import interpretar_decisao_ferramenta; texto = 'isso não é um json'; print(interpretar_decisao_ferramenta(texto))"

============================================================

Esse teste verifica se o chat principal consegue usar o RAG por tool calling.

O fluxo esperado é:

1. usuário faz uma pergunta sobre os materiais;
2. a LLM decide usar consultar_material_rag;
3. o main.py chama executar_ferramenta();

5. o RAG recupera chunks relevantes;
6. o Gemma responde com base nos trechos;
7. a resposta aparece no chat;
8. a chamada é registrada em logs/tool_calls.jsonl.

Comando para iniciar o chat:

python -m src.main

Mensagem para testar:

Explique regressão logística com base nos materiais

Resultado esperado:

O JARVIS deve responder com uma explicação baseada nos materiais carregados em data/materiais.

Também é aceitável aparecer mensagem de timeout se a API do Gemma estiver lenta, desde que o programa não quebre com traceback.

Depois testar:

Quais são os principais pontos do material sobre regressão logística?

Resultado esperado:

O JARVIS deve usar consultar_material_rag novamente e responder com base nos documentos.
